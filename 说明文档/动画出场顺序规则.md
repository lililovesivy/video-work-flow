# 动画出场顺序规则

## 核心原则

动画不是让画面动起来，而是控制观众的阅读顺序。

每个素材视频都必须先拆清楚：

1. 这句话先让观众知道什么问题
2. 再让观众看到什么证据或人物
3. 接着解释证据或人物的哪一个重点
4. 最后给出什么判断或结论

禁止所有元素一次性出现。每个元素必须有明确的出场理由。

## 默认出场逻辑

### 1. 先给主题

先出现标题、问题句或章节标签，让观众知道这一页讲什么。

适合元素：

- 大黑标题
- 顶部素材编号
- 一句短问题
- 核心关键词

推荐动画：

- 标题淡入上移
- 页面轻微推近

### 2. 再给主体

主题出现后，再出现画面的主证据或主角色。

适合元素：

- 新闻截图
- 法条截图
- 判决截图
- 人物抠图
- 房产证、车辆、合同等核心实物
- 主体关系卡片

推荐动画：

- 截图轻微滑入
- 人物淡入
- 卡片淡入
- 文件轻微推近

### 3. 再给连接关系

主体稳定后，再出现箭头、虚线、连线，告诉观众主体之间是什么关系。

适合元素：

- 箭头
- 虚线
- 资金流向
- 房产流转方向
- 人物关系连接线

推荐动画：

- 箭头/虚线生长
- 关系线分段出现

### 4. 再给解释标签

关系建立后，再给标签解释主体含义。标签不能先于主体出现，否则观众不知道它指向谁。

适合元素：

- 人物身份标签
- 心理状态标签
- 风险点标签
- 条件判断标签
- 关键词黄标

推荐动画：

- 标签贴出
- 条件逐条点亮
- 黄色高亮扫过

### 5. 再强调关键证据

当观众已经知道主体和关系后，再对证据里的关键句、关键数字、关键比例做高亮。

适合元素：

- 法条关键句
- 判决关键段
- 金额
- 比例
- 年龄
- 时间
- 风险关键词

推荐动画：

- 黄色高亮扫过
- 红框描出
- 数字短促递增
- 局部放大

### 6. 最后给结论

结论必须最后出现，用来收束画面。结论不能一开始就压出来，否则会抢走证据和推理过程。

适合元素：

- 底部黑底白字结论条
- 红色结论大字
- 法院认定
- 风险判断

推荐动画：

- 黑底结论条压入
- 红色结论压入
- 重点词轻微放大

## 常见画面类型的推荐顺序

### 长文本法条页

1. 标题出现：先说明这一页讲的法律问题
2. 文件截图出现：让观众知道有原始依据
3. 截图轻微推近：稳定视线
4. 关键句高亮：只标出最重要的 1-3 句
5. 旁侧解释标签出现：把法条翻译成人话
6. 结论条出现：给出本句口播对应判断

### 原素材新闻页

1. 标题出现：先抛出新闻或争议点
2. 新闻截图出现：截图作为主视觉
3. 新闻标题或关键段高亮
4. 辅助示意图出现：比例要小，只辅助理解
5. 短标签解释新闻重点
6. 结论条出现

### 人物标签页

1. 标题出现：先说明人物问题
2. 人物淡入：人物成为视觉锚点
3. 人物边缘描边显现：把人物从背景中分离
4. 标签逐个贴出：按口播顺序出现
5. 虚线连接标签和人物
6. 关键风险词或结论最后出现

### 证据判断页

1. 标题出现：说明判断主题
2. 主证据或主节点出现
3. 次级节点依次出现
4. 箭头按因果或时间顺序生长
5. 判断条件逐条点亮
6. 红色结论压入

### 资金/房产流转页

1. 标题出现：说明这一步流转讲什么
2. 起点主体出现
3. 中间标的物出现
4. 箭头从起点生长到中间
5. 终点主体出现
6. 箭头从中间生长到终点
7. 金额、比例、登记名义等标签出现
8. 底部结论条出现

### 对比判断页

1. 标题出现
2. 左侧观点或主张出现
3. 右侧事实或法院认定出现
4. 差异点高亮
5. 判断依据出现
6. 结论压入

## 时间分配建议

一个 `5-6` 秒素材的默认节奏：

- `0.0-0.8s`：标题和主题出现
- `0.8-2.0s`：主素材或人物出现
- `2.0-3.4s`：关系、箭头、标签依次出现
- `3.4-4.6s`：关键证据高亮
- `4.6-6.0s`：结论出现并停留

复杂素材最多延长到 `7-8` 秒，但不要靠拖慢动画解决信息过载。

## 复查标准

导出后必须抽查关键帧：

- 早期帧：标题是否先出现，主体是否没有过早堆满
- 中期帧：主体、箭头、标签是否按逻辑顺序出现
- 结尾帧：所有元素是否完整可读、没有遮挡
- 全程：标签和箭头是否没有穿过人物、截图和关键文字
