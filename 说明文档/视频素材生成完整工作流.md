# 视频素材生成完整工作流

更新时间：2026-06-17

## 1. 当前工作流定位

这套系统现在做的，不是“整条视频一键成片”，而是：

**把口播文稿拆成一批批可审核、可编辑、可直接转视频的页面素材。**

最终产物分两层：

- 可编辑 `PPT`
- 每个素材编号对应的独立视频

主时间线拼接仍然放在剪映、Premiere、Final Cut 或 DaVinci Resolve 中完成。

---

## 2. 和最初版本相比，发生了什么变化

最初版本的核心逻辑是：

- 每批只推进 `2` 个素材
- 先生成静态 `PNG` 预览图
- 用户确认 `PNG` 后再做视频
- 更偏“图片审核链路”
- 背景、强调、版式规则还比较粗

现在的版本已经改成：

### 2.1 生产链变化

- 每批默认推进 `4` 个素材，不再是 `2` 个
- 默认先生成可编辑 `PPT`，不再把 `PNG` 预览作为正式交付中间态
- `PPT` 通过后，直接从 `PPT` 生成视频
- `PNG` 现在只作为内部自检或快速预览工具，不再是正式审核中心

### 2.2 并行方式变化

- 视觉审核表通过后，默认同时打开 `2` 个 sub-agent
- 一次并行完成 `4` 页 `PPT`
- 每个 sub-agent 负责 `2` 页
- 不再采用单 agent 串行一页页做完的方式

### 2.3 页面任务定义变化

- 从“一个页面尽量装更多信息”改成“一个页面只承担一个阅读任务”
- 如果同一页里同时出现“结论 + 因素”“规则 + 举例”“转场 + 说明”，应主动拆页
- 转场页默认极简，只保留 `标题 + 钩子句 + 一个主视觉`
- 流转页默认先讲 `起点 -> 动作 -> 终点`，不先写抽象总结

### 2.4 视觉风格变化

- 从偏通用短视频包装，收敛到“法律讲义内容页”
- 重点不再靠堆高亮和堆色块制造存在感
- 色块只服务于真正的主结论、风险点、关键停顿词
- “满页都是重点”的写法已经被废弃

### 2.5 页面文案规则变化

- 页面上禁止出现内部工作编号，如 `A017`、`A18`
- 页面上禁止出现“占位”“本页只讲”“规则提示”“制作说明”这类制作侧文案
- 画面文案必须直接面向观众

### 2.6 资产调用规则变化

- 优先调用 `已抠图ui`
- 如果 `已抠图ui` 没有对应素材，必须在设计表里明确提示
- 不再默认临时乱找替代图

### 2.7 视频生成路径变化

- 已经确认：不必先把 `PPT` 固化成图片才能做视频
- 当前固定链路是：

**逐句审核表 -> 可编辑 PPT -> 用户修改/确认 -> 直接生成视频**

### 2.8 风格确认方式变化

- 不再默认一上来就连续生成整批正文页面
- 每次新项目开始时，先设计 `4` 个开场页供用户选择
- 用户先从这 `4` 个开场页里确认配色、气质、版式方向
- 风格一旦确认，后续页面必须继承同一套风格再继续批量生成

## 3. 当前标准生产链

### 第 1 步：文稿拆句

把完整口播拆成逐句编号，例如：

- `S01`
- `S02`
- `S03`

每一句都要判断：

- 是否需要视觉素材
- 为什么需要
- 如果需要，应该由哪一种视觉页面承担

### 第 2 步：生成逐句视觉审核表

审核表必须写清：

- 句序
- 口播原句
- 是否需要视觉素材
- 判断原因
- 素材编号
- 视觉模式
- 视觉素材语言描述
- 建议动画效果
- 素材来源链接建议
- 用户是否需要截图或下载
- 建议时长

审核表阶段就要完成这些判断：

- 这一句到底值不值得做素材
- 这一页的唯一核心内容是什么
- 这一页是否已经超载，应该提前拆页
- 这一页有没有真实外部素材依赖

### 第 3 步：用户审核逐句表

只有审核表通过，才进入 `PPT` 生产。

如果用户已经明确改过某类页面结构，后续同类页面必须继承最新改法，不能继续沿用被推翻的旧模板。

### 第 4 步：每批推进 4 个素材进入 PPT

当前默认批次大小是 `4`。

执行方式固定为：

- 同时打开 `2` 个 sub-agent
- 每个 sub-agent 负责 `2` 页
- 一次合成 `4` 页可编辑 `PPT`

### 第 5 步：先做 4 个开场页定风格

每次新文稿开始，不直接进入整批正文页生产。

必须先做：

- `4` 个开场页方案
- 这 `4` 页默认承担“标题页 / 提问页 / 总答案页 / 第一转场页”一类开场任务
- 重点不是一次把信息讲完，而是让用户先选定：
  - 配色
  - 背景处理方式
  - 标题层级
  - 高亮方式
  - UI 放置习惯
  - 整体气质

只有用户确认开场页方向后，后续页面才继续生成。

### 第 6 步：再交付可编辑 PPT 正文批次

`PPT` 是当前正式审核中心。

用户可以直接修改：

- 标题
- 黄标
- UI
- 截图框
- 对齐
- 构图
- 配色

正文批次默认继续按每批 `4` 页推进，但必须继承已确认的开场风格。

### 第 7 步：PPT 通过后直接生成视频

视频阶段不再另起一套视觉风格，只做两件事：

- 按确认后的页面结构补动画
- 导出每个素材编号对应的独立视频

### 第 8 步：当前 4 个满意后再进入下一批

不能一口气做完整条视频的所有素材。

固定节奏是：

**当前 4 个通过 -> 再做下一批 4 个**

## 4. 当前页面设计原则

### 4.1 一页只做一件事

一页默认只解决一个理解瓶颈。

禁止把这些任务混在同一页里：

- 问题
- 结论
- 判断因素
- 转场说明
- 流转关系
- 行动建议

### 4.2 一页只保留一个主重点

一页里只能有一个真正的主强调。

优先顺序通常是：

1. 主结论
2. 主问题
3. 主反转
4. 主行动建议

其余信息退回普通文字、位置层级、留白层级，不靠第二块、第三块高亮去抢注意力。

### 4.3 色块使用原则

- 色块不是装饰
- 色块不是默认配置
- 色块只给真正需要停住看的内容

普通节点、结构标签、解释句、辅助说明，优先不用色块。

### 4.4 页面中不出现内部编号

以下内容只允许留在脚本、审核表、文件名中：

- `A01`
- `A17`
- `A24`
- 页码条
- 黑色任务标签条

这些都不允许进入最终页面。

### 4.5 页面文案面向观众

禁止出现：

- 占位
- 本页只讲
- 规则提示
- 制作说明
- 这里放
- 这一页要表达

页面文案必须是观众能直接读懂的内容。

## 5. 当前视觉风格基线

### 5.1 风格方向

当前方向不是“花哨短视频包装”，而是：

**抖音普法内容页 / 法律讲义页**

关键词：

- 米白纸感
- 黑字主标题
- 克制的米金强调
- 少量风险红
- 暖白卡片
- 截图、法条、人物、合同等真实证据感元素

但每个新项目的具体风格，不再靠我直接延续旧稿默认决定，而是先通过 `4` 个开场页由用户选定。

### 5.2 当前背景处理

当前成熟做法是：

- 使用米白纸质感背景
- 背景可见度压到约 `33%`
- 通过白色蒙层把纹理变成“底气氛”，而不是主视觉噪音

背景不再是硬性“一张图走到底”的规则，但同一批里应保持系列统一。

### 5.3 当前高亮和边框处理

- 荧光黄方案已废弃
- 当前强调色改为米金系
- 卡片边框采用低对比暖灰棕，不用死黑描边
- 重点靠内容层级，不靠高饱和度

## 6. 当前资产调用规则

### 6.1 UI 调用优先级

优先级固定为：

1. `已抠图ui`
2. 用户已上传的现成素材
3. 真实截图或法条页
4. 如无合适资产，在设计表中明确提示缺失

### 6.2 缺图处理原则

如果 `已抠图ui` 没有相应图片：

- 不能装作已有
- 不能随手找不相干替代
- 必须在视觉审核表或设计说明中明确提示用户

### 6.3 真实素材优先级

如果有以下内容：

- 新闻截图
- 法条截图
- 判决截图
- 合同/文书截图

它们优先作为主视觉，示意 UI 只能辅助，不得反客为主。

---

## 7. 当前动画执行规则

动画只负责阅读引导，不负责重做版式。

默认出场顺序：

1. 主题
2. 主体
3. 关系
4. 标签
5. 证据强调
6. 结论

当前常用动画词汇：

- 标题淡入上移
- 黄色高亮扫过
- 截图轻微推近
- 卡片依次出现
- 连线/箭头生长
- 标签贴出
- 时间线推进
- 左右对比亮起
- 数字短促递增
- 红色结论压入

视频阶段必须遵守：

- 不改 `PPT` 已确认的主结构
- 不改 `PPT` 已确认的主配色
- 不因动画新增遮挡

## 8. 当前 QA 规则

### 8.1 PPT 自检

交给用户前必须自检：

- 是否只有一个阅读任务
- 是否只有一个主重点
- 是否还有内部编号或制作标签
- 是否还有制作侧文案
- 是否有遮挡
- 是否有裁切
- 是否有错字
- 是否有图形变形
- 高亮块大小是否和文字相称
- 文字是否正好落在高亮几何中心

### 8.2 视频自检

视频导出后必须抽查：

- 首帧
- 中间帧
- 尾帧

重点检查：

- 是否新增遮挡
- 是否有裁切
- 是否有缩放后变形
- 是否和 `PPT` 版式不一致

---

## 9. 当前正式固定下来的流程

现在的唯一正式链路是：

**逐句视觉审核表 -> 先出 4 个开场页定风格 -> 用户确认风格 -> 4 页可编辑 PPT 并行生成 -> 用户直接修改/确认 PPT -> 直接生成视频 -> 下一批 4 页**

如果后续还有新规则，应该优先改这条主链，而不是回退到旧的：

- `2` 页一批
- `PNG` 为中心
- 单 agent 串行
- 一页多重点
- 先不定风格就直接往后连续生成
