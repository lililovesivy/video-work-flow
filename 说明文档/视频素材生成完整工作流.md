# 视频素材生成完整工作流

更新时间：2026-06-13

## 1. 工作流定位

这个文件夹搭建的是“视频素材生成工作流”，不是整条短视频的一键成片系统。

目标是把法律、商业、财经类口播文稿拆成若干视觉理解点，再按批次生成可以插入剪映、Premiere、Final Cut 或 DaVinci Resolve 主时间线的独立短视频素材。

核心原则：

- 文稿驱动素材，不是素材反过来主导文稿。
- 一个素材只解决一个理解瓶颈。
- 每个素材编号对应一个独立 MP4。
- 每次只推进 2 个素材，用户确认后再继续。

## 2. 标准生产链路

| 阶段 | 动作 | 输出 |
|---|---|---|
| 1 | 读取口播文稿 | 完整文稿理解 |
| 2 | 逐句拆解 | S01、S02、S03 等逐句编号 |
| 3 | 判断每句是否需要视觉素材 | 逐句文稿-视觉素材对照表 |
| 4 | 用户审核对照表 | 确认需要生成哪些素材 |
| 5 | 每批选择前 2 个素材 | A01/A02 或下一批两个素材 |
| 6 | 先生成静态预览图 | PNG 预览图 |
| 7 | 用户确认预览图 | 修改到无遮挡、无误读 |
| 8 | 再生成视频 | 每个素材一个独立 MP4 |
| 9 | 抽查视频关键帧 | 首帧、中间帧、尾帧或更多检查帧 |
| 10 | 用户满意后进入下一批 | 下一批 2 个素材 |

## 3. 逐句审核表规则

逐句表必须精确到每一句话。每一句都要判断：

- 是否需要视觉素材
- 为什么需要或不需要
- 对应素材编号
- 视觉模式
- 视觉素材语言描述
- 建议动画效果
- 素材来源链接建议
- 用户是否需要下载或截图
- 配色与质感提示
- 建议时长

重要规则：

- 不再使用“并入上一素材”。
- 如果某句只是在剪辑中延续上一画面，标记为“不需要素材”，备注写“剪辑中延续上一素材，不单独生成”。
- 文案中提到具体公司、品牌、机构、法条、新闻、公告、网页时，在审核表中提供来源链接，不默认代下载图片。
- 用户需要的新闻图、法条页、logo、网页截图，由用户自行截图或下载后放入素材目录。

## 4. 独立 MP4 规则

每个视觉素材编号必须对应一个独立 MP4 文件。

正确方式：

- `A01-主题.mp4`
- `A02-主题.mp4`
- `A03-主题.mp4`

禁止：

- 把 A01 和 A02 拼成一个视频。
- 在一个素材视频里塞多个不同视觉画面。
- 视频内容和审核表里的视觉描述不一致。

如果画面主题、结构、结论发生变化，就必须拆成新的素材编号。

## 5. 当前视觉风格

目标风格是抖音普法/商业解释内容页，参考“法律之盐”一类账号的内容页逻辑。

关键词：

- 白底或浅灰纸感背景
- 黑色大标题
- 黄色高亮
- 红色结论
- 法条、新闻、网页、判决截图
- 真实人物抠图或剪影
- 证据结构页
- 内容页质感

不要做成：

- 高级商务海报
- 公司宣传片
- 过度信息图
- 赛博霓虹
- 娱乐化包装
- 抽象图标堆叠

## 6. 构图规则

构图优先级：

1. 原素材可信度
2. 重点结论可读性
3. 观众阅读顺序
4. 人物或示意图辅助理解
5. 画面装饰

长文本处理：

- 长文本来自法条、判决、新闻、合同，就优先作为“证据原件”展示。
- 不要把整段长文本重排成设计文字。
- 只高亮 1-3 句关键内容。
- 可裁切、倾斜、局部放大，保留证据感。

原素材截图处理：

- 有新闻截图、法条截图、判决截图时，截图应成为主视觉。
- 原素材通常占画面 45%-65%。
- 示意图只能辅助，不能抢主体。
- 重点段落使用黄色高亮、红框、下划线。

人物处理：

- 人物必须写实或剪影，不使用卡通风。
- 需要能看出性别和年龄范围。
- 标签放在人物外侧，用虚线或细线连接。
- 标签线不能穿过人物脸部、关键文字或截图高亮区。

## 7. 动画能力和使用逻辑

当前稳定可用的动画包括：

- 标题淡入上移
- 黄色高亮扫过
- 截图轻微推近
- 截图滑入或压入
- 卡片依次出现
- 箭头或虚线生长
- 标签贴出
- 人物淡入
- 人物轻微推近
- 人物边缘描边显现
- 数字短促递增
- 条件逐条点亮
- 左右对比亮起
- 红色结论压入
- 黑底白字结论条压入

动画不是一次性全部出现，而是控制观众阅读顺序。

默认顺序：

1. 主题：标题、问题句、章节标签
2. 主体：新闻截图、法条截图、人物、房产证、车辆、合同
3. 关系：箭头、虚线、资金流、房产流转线
4. 解释：人物标签、风险标签、条件标签、黄标
5. 证据强调：黄色高亮、红框、局部放大、数字递增
6. 结论：黑底白字结论条、红色结论

5-6 秒素材默认节奏：

| 时间 | 内容 |
|---|---|
| 0.0-0.8s | 标题出现 |
| 0.8-2.0s | 主体素材出现 |
| 2.0-3.4s | 箭头、关系、标签依次出现 |
| 3.4-4.6s | 关键证据高亮 |
| 4.6-6.0s | 结论出现并停留 |

## 8. QA 自检规则

每次生成预览图和视频后，必须先自检再提交给用户。

预览图检查：

- 文字、图形、截图、人物、箭头、标签是否互不遮挡
- 关键文字是否完整可读
- 文字是否被裁切、压边、覆盖
- 图形是否变形
- 具体物件是否能识别，例如车、房产证、新闻截图
- 人物性别、年龄、身份是否清楚
- 图文关系是否对应文案

视频检查：

- 每个素材是否独立导出 MP4
- 动画是否按主题、主体、关系、标签、证据、结论依次出现
- 首帧、中间帧、尾帧是否无遮挡
- 移动、缩放、淡入后是否没有产生新遮挡
- 视频阶段是否没有擅自改变已确认的预览图主结构和配色

如果发现遮挡、错字、裁切、变形、性别年龄不清、图文不对应，必须先返工，不能直接交给用户复查。
