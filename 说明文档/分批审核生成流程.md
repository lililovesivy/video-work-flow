# 分批审核生成流程

## 目标

把素材生成流程改成低风险、可审核、分批推进的方式，避免一次性生成过多无效素材。

## 执行规则

1. 先逐句拆解口播文稿
2. 为每一句判断是否需要视觉素材
3. 如果需要，先写：
   - 视觉任务类型
   - 视觉素材语言描述
   - 建议动画效果
   - 素材来源链接建议
   - 是否需要用户截图或下载
   - 建议时长
4. 如果句子里出现具体公司、品牌、机构名称，优先在表格中提供官网、品牌页、公告页或可信新闻页链接，不默认代下载或截图
5. 需要真实 logo、新闻图、法条页、公告页时，由用户根据链接手动截图或下载后再进入画面设计
6. 先输出逐句对照表给你审核
7. 审核通过后，先设计 `4` 个开场页给用户选择，用来确认这一条视频的配色、背景、标题层级和整体气质
8. 只有开场页风格确认后，才默认推进前 `4` 个素材进入可编辑 `PPT` 阶段
9. 进入 `PPT` 阶段后，应同时打开 `2` 个 sub-agent，并行生成 `4` 页可编辑版面；每个 sub-agent 负责 `2` 页，不再让单个 agent 串行生成全部版面
10. 先提交这 `4` 页可编辑 `PPT` 给用户修改和审核
11. `PPT` 通过后，直接按素材编号分别补动画并生成独立视频，不再默认额外生成静态 `PNG` 预览链路
12. 当前这一批视频满意后，再继续下一批 `4` 个素材

## 独立素材视频规则

- 视觉审核表里的每个 `素材编号` 都必须输出为一个独立 MP4 文件
- 每个素材页只允许有一个核心表达，不能在一个页内同时承担“问题 + 结论 + 关系图”多个任务
- 每个素材页默认只放一个最契合的辅助 UI；没有必要时可以不放
- 如果一个素材页同时包含“规则结论”和“判断因素”，应默认考虑拆成两页，而不是先塞满一页再交给用户修改
- 例如 `A01` 是 `5s`，就只导出 `A01-主题.mp4`；`A02` 是另一个 `5s`，就另导出 `A02-主题.mp4`
- 禁止把 `A01` 和 `A02` 拼在同一个视频里
- 禁止在一个素材视频内部塞入多个不同视觉画面；只要画面主题、结构或结论改变，就拆成新的素材编号
- 如果口播只是继续解释上一张画面，不需要新素材，审核表里标 `否`，备注写 `剪辑中延续上一素材，不单独生成`
- 口播主时间线由剪映、Premiere、Final Cut 或 DaVinci Resolve 负责组装；素材视频只做可插入的短片段，不承担整条视频剪辑

## 为什么这样做

这样做的核心好处有三个：

- 防止前期方向跑偏后整批返工
- 先确认画面信息结构，再确认动画表达
- 把“文稿理解错误”和“动效实现问题”拆开处理

## 每一批的产出顺序

### 第 1 阶段：审核表

输出逐句文稿和视觉素材对照表，供人工确认：

- 哪一句需要素材
- 为什么需要
- 应该生成什么样的视觉画面
- 适合什么动画方式
- 是否需要真实截图、logo、新闻页或法条页
- 对应来源链接在哪里

### 第 2 阶段：前 4 个素材可编辑 PPT
只生成可编辑 `PPT` 页面，不加动画，重点确认：

- 这一阶段开始前，必须先完成 `4` 个开场页风格提案，由用户先选定这一条视频的视觉方向
- 开场页承担的是“定风格”任务，不承担整条视频的信息压缩任务
- 一旦用户选定开场页方向，后续正文页必须继承同一套配色、背景处理、高亮习惯和版式气质
- 如果开场页风格还没定，就不要直接往后连续生成正文页

- 视觉审核表一旦通过，默认同时启动 `2` 个 sub-agent
- 每个 sub-agent 负责 `2` 页，合计一次产出 `4` 页可编辑版面
- 不再使用“一个 agent 连续做两页、再继续做下一页”的串行方式
- 两个 sub-agent 的责任边界要提前划清，避免改同一脚本或互相覆盖输出

- 生成前先参考 `说明文档/样片构图规律-长文本原素材人物.md`，决定是长文本法条页、证据判断页、人物标签页，还是原素材主导页
- 背景和法律名词 UI 图先参考 `说明文档/法律名词ui使用规则.md`，优先使用 `已抠图ui` 缓存图；背景不再强制统一为 16:9 通用背景，应按当前页面主题选择
- 一旦某页选定了背景，所有视觉元素都必须压在该背景主区域内，不得越出当前背景边缘
- 每一页只保留一个核心内容，不为“信息更满”额外堆第二结论或第二信息层
- 每一页默认只放一个最契合的 UI 辅助图，避免一个页内堆多个 UI 图标
- 但“只放一个 UI”不等于“只允许一个结构节点”；人物关系、资产流转、判断因素这类页面可以有多个节点，只要它们共同服务于同一个核心表达
- 转场页默认极简，不主动补长解释段，优先使用 `标题 + 钩子句 + 一个主视觉`
- 流转页默认使用 `起点 -> 动作 -> 终点` 的节点化表达，而不是先写抽象概念
- 页面中禁止出现“占位”“本页只讲”“规则提示”“这里用”这类制作侧提示语
- `PPT` 是否为可编辑对象，文字、黄标、截图框、UI 图是否能直接修改
- 信息结构是否对
- 标题和标签是否对
- 关系、顺序、对比逻辑是否对
- 风格是否符合当前项目规范
- 配色是否统一、克制、协调
- 画面质感是否稳定，不出现跳色和廉价感
- 文字、图形、截图、人物、箭头、标签之间是否互不遮挡
- UI 图是否压住标题、主证据、主句子或底部结论条
- 关键文字是否完整、可读，没有被裁切或压到画面边缘
- 图形是否没有变形，具体物件是否能一眼识别

生成后必须先自检再交付审核；如果出现遮挡、裁切、错字、变形、人物性别/年龄不清、图文关系不对应，必须先返工，不得把问题 `PPT` 直接交给用户复查。

### 第 3 阶段：前 4 个素材视频

在 `PPT` 确认后，再直接补动画，重点确认：

- 每个素材编号是否导出了独立 MP4
- 视频内部是否只表达审核表对应的单一视觉画面
- 动画前先参考 `说明文档/动画出场顺序规则.md`，按主题、主体、关系、标签、证据强调、结论的顺序拆解元素
- 动画节奏是否干净
- 动画是否帮助理解而不是干扰理解
- 时长是否合适
- 是否适合插入口播节奏
- 视频阶段不得擅自更换 `PPT` 已确认的主配色体系和主构图
- 动画增强层级，但不能靠高饱和颜色制造存在感
- 动画过程中是否始终避免文字、图形、截图、人物、箭头、标签互相遮挡
- 入场、移动、缩放后关键文字是否仍然完整可读

视频导出后必须抽查首帧、中间帧、尾帧；如果任一关键帧出现遮挡、裁切、变形或图文不对应，必须先返工再提交。

### 第 4 阶段：进入下一批

只有当前 `4` 个素材视频确认满意，才进入下一批 `4` 个素材。

## 当前默认动画词汇

后续在逐句对照表里，动画建议优先使用以下可控词汇：

- `标题淡入上移`
- `黄色高亮扫过`
- `截图轻微推近`
- `卡片依次出现`
- `连线/箭头生长`
- `标签贴出`
- `时间线推进`
- `左右对比亮起`
- `数字短促递增`
- `曲线描出`
- `红色结论压入`
- `人物/背景轻微视差`

避免一开始就写成复杂镜头语言，先保证信息表达准确。

## 当前配色执行规则

- 所有批次沿用同一套主背景、主文字、边框和强调色
- 单个素材内不使用超过两种强调色
- 不使用荧光色、高纯度撞色、强娱乐化渐变
- `PPT` 确认后，视频版本默认继承同一配色和主版式，不重新换风格

## 新增风格回读规则

- 用户对某一批 `PPT` 做出大幅结构改动后，后续批次生成前必须先回读这些修改，提炼新的构图和信息拆分规律
- 后续批次不得继续沿用已被用户推翻的旧模板写法
- 如果用户已经把一页拆成两页，后续遇到同类结构时应主动预拆，而不是再次等用户返工

## 品牌素材规则

- 文案中提到具体公司或品牌时，优先在审核表里提供来源链接
- 优先顺序：官网公开素材页 > 官网公开页面 > 公司公告/年报/新闻稿 > 可信新闻或数据页面
- 不再默认由我下载或截图 logo；用户手动下载或截图后再用于 `PPT` 和视频
- 进入公开视频前，注意商标和品牌使用合规

## 样片风格规则

- 画面优先像“短视频内容页”，不是“高级商务海报”
- 白底页使用黑字、黄标、红色结论，保持讲义感
- 截图页优先保留真实页面质感，用高亮和标签引导阅读
- 暗场产业页只用于芯片、AI、航空、金融、制造等题材，避免赛博霓虹
- 动画只做阅读引导，不做设计秀

## 当前结论

后续执行顺序固定为：

**逐句审核表 -> 先出 4 个开场页定风格 -> 2 个 sub-agent 并行产出前 4 页可编辑 PPT -> 前 4 个独立动画视频 -> 下一批 4 个。**
- 最终页面不得出现工作阶段自用编号，如 `A017`、`A18`、页码条、任务标签条；这些只用于内部生产，不进入用户可见页面。
- 色块使用遵循“少而准”原则：一页里如果已经有一个主结论高亮，其余信息优先用字重、位置、留白区分，避免满页高亮导致失去重点。
