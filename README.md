# 视频素材生成工作流

这是一个面向法律、商业、财经类口播视频的素材生产系统。

目标不是一键生成整条视频，而是把口播文稿拆成可审核、可编辑、可直接转视频的一批批页面素材。

## 当前唯一正式链路

- 逐句拆解口播文稿，先生成逐句视觉审核表。
- 每个素材编号对应一个独立视频，不拼多个素材到同一个文件。
- 每批默认推进 `4` 个素材，不再是旧版的 `2` 个。
- 审核表通过后，先并行生成 `4` 页可编辑 `PPT`，不再以静态 `PNG` 预览作为正式审核中心。
- 用户直接修改和确认 `PPT` 后，再从 `PPT` 直接生成视频。
- 页面必须遵守“一页一个阅读任务、一个主重点”，禁止内部编号、制作提示语和满页高亮。

## 当前重点规则

- 文稿驱动素材，不是素材反过来主导文稿。
- 一页只解决一个理解瓶颈。
- 色块只给真正的主结论、风险点或关键停顿词使用。
- 页面上不允许出现 `A01`、`A17` 这类内部工作编号。
- 优先使用 `已抠图ui`，缺图时必须明确提示，不能乱找替代。
- 有新闻、法条、判决、合同等真实截图时，原素材优先作为主视觉。
