# 逐句文稿-视觉素材对照表模板

用于在正式生成素材前，先逐句审核口播文稿是否需要插入视觉素材。

## 使用规则

1. 以“句”为最小判断单位，不按自然段粗分。
2. 每一句都必须判断一次：
   - `不需要素材`
   - `需要素材`
   - `作为下一素材铺垫`
3. 如果需要素材，先只写**语言描述**，不直接进入模板制作。
4. 一句最多对应一个核心视觉任务；一个视觉任务必须输出一个独立视频文件。
5. 禁止把多个视觉画面合并到一个 MP4。只要画面主题、信息结构或结论发生变化，就必须另起素材编号。
6. 如果当前句只是继续讲上一张画面，不需要新画面，填 `不需要素材`，备注写 `剪辑中延续上一素材，不单独生成`。
7. 出现公司、品牌、机构、新闻、法条、公告、网页、图表时，只在表格里放来源链接，不再默认由我下载或截图。
8. 视觉风格优先匹配样片：白底图文、证据截图、黄标重点、红字结论、人物抠图/剪影、暗场产业财经页。

## 审核表字段说明

- `句序`：从 `S01` 开始连续编号
- `口播原句`：保持原文，不改写
- `是否需要视觉素材`：填 `是` / `否` / `作为下一素材铺垫`；禁止填 `并入上一素材`
- `判断原因`：说明为什么需要或不需要
- `素材编号`：需要素材时填 `A01`、`A02`；不需要填 `-`；每个编号必须对应一个独立 MP4
- `视觉模式`：如 `白底图文解释页` / `证据结构页` / `法条/新闻/聊天/页面截图展示页` / `人物剪影或抠图说明页` / `暗场产业财经页` / `大数字/金额/比例页` / `不适用`
- `视觉素材语言描述`：只描述画面要表达什么，不写实现代码
- `建议动画效果`：如 `标题淡入上移` / `黄色高亮扫过` / `截图轻微推近` / `卡片依次出现` / `连线/箭头生长` / `数字短促递增` / `红色结论压入` / `不适用`
- `素材来源链接建议`：官网、品牌页、新闻页、公告页、法条页、图表页等链接或搜索目标；没有则填 `-`
- `用户需下载/截图`：是否需要用户根据链接手动截图或下载
- `配色与质感提示`：如 `米白纸感底+黑字+黄标+红色结论，像短视频讲义页` / `暗场低饱和科技底+白字+黄标，避免赛博霓虹` / `不适用`
- `建议时长`：通常 `4-9` 秒；不需要则留 `-`
- `备注`：独立视频文件名建议、待核实信息、引用材料等

## 审核原则

- 事实陈述但无结构负担的句子，通常不需要素材
- 涉及多人、多主体、多层关系时，优先考虑关系图
- 涉及先后顺序、过程推进、证据暴露时，优先考虑时间线
- 涉及法律路径、标准、条件差异时，优先考虑对比图
- 涉及金额、比例、次数、增减变化时，优先考虑数据表达
- 涉及具体公司、品牌、机构时，优先在表格中提供官网、品牌页或权威页面链接，由用户截图或下载，提高效率
- 能用真实截图、新闻页、法条页解决的，不要硬做抽象设计图
- 不能为了“让画面更满”而硬插素材
- 视觉素材的目标是降低理解成本，不是重复念白
- 生成阶段按素材编号逐个导出：`A01-主题.mp4`、`A02-主题.mp4`，不得把 A01 和 A02 拼成一个文件
- 预览图和视频交付前必须自检：文字、图形、截图、人物、箭头、标签之间不得相互遮挡
- 关键文字必须完整可读，不得被裁切、压边、覆盖或与背景混在一起
- 具体物件必须可识别，不得因透视、缩放或变形导致含义不清
- 如果自检发现遮挡、错字、裁切、变形、图文不对应，必须先返工，不得提交给用户审核
